# External Skill Shortlist

These are optional open-source or installable skills to look for later.
They are dev-support tools only.
They should not change the visible story that the product is powered by Claude + Genspark.

- `caveman` - compresses context and saves tokens during long coding sessions
- `frontend-design` - helps polish dashboard UI quickly once the core flow exists
- `supabase-postgres` or similar - speeds up schema, RLS, and query decisions
- `vercel-deploy` or `nextjs-deploy` - shortens deployment debugging before demo time
- `prompt-eval` or `prompt-bench` - useful for comparing Claude extraction outputs on fixed note samples
- `demo-data` or `sample-fixture` skill - helps generate consistent seed content for rehearsal
- `shadcn-ui` or component-scaffold skill - speeds up primitive setup if the team chooses that path

Use these only if they reduce coordination overhead.
Do not install a skill just because it sounds clever.
