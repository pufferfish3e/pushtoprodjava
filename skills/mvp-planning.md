# MVP Planning Skill

## Use this when

- deciding what the team should build first
- cutting scope during the sprint
- checking whether a feature belongs in the demo or the roadmap

## Goal

Keep the team focused on the smallest product that still wins the room.

## Hard MVP

- notes paste
- Claude extraction
- save tasks
- dashboard visibility
- one good Genspark query

## Preferred order

1. Auth and data path are usable.
2. Extraction returns validated JSON.
3. Tasks render in a clear event view.
4. Cross-event overview exists.
5. Genspark query works.
6. Demo polish and fixtures get tightened.

## First cut candidates when behind

- export formatting
- calendar-heavy views
- scheduled agents
- Telegram alerts
- audio transcription

## Team timing guidance

By roughly the halfway point, the team should already be able to run:

`paste notes -> extract -> see tasks on screen`

If that is not working, stop adding features and fix the core loop.

## Acceptance checklist

- every teammate can state the MVP in one sentence
- no active workstream is drifting into roadmap-only features
- the demo path is testable before polish begins
